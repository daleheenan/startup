## Archive Reasons Reference

When archiving, use one of these standard reasons:

| Reason Code | Description |
|-------------|-------------|
| `NEVER_APPLIED` | Score 0 after 10+ agent tasks |
| `LOW_VALUE` | Score < 3 after 6+ months |
| `ONE_TIME` | Only applied to specific one-time situation |
| `SUPERSEDED` | Replaced by better lesson |
| `OBSOLETE` | Technology/pattern no longer used |
| `VAGUE` | Not specific or actionable enough |
| `DUPLICATE` | Merged into another lesson |

## Revival Process

To revive an archived lesson:

1. Find the lesson in this file
2. Verify it's still relevant to current codebase
3. Update any obsolete references
4. Reset application score to 0
5. Move to appropriate active lesson file
6. Add note: `**Revived from archive**: YYYY-MM-DD, originally archived because [reason]`
